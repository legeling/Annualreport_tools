# 设计文档

## 信息架构

应用采用三栏工作台结构：

1. 左侧文档管理区
   - 当前导入的年报链接表
   - 本地 PDF 文件夹入口
   - 年份、板块、行业筛选器

2. 中部年报列表区
   - 顶部搜索和主操作按钮
   - 状态统计卡片
   - 年报表格
   - 列表状态分段控件

3. 右侧详情与处理区
   - PDF 预览
   - 打开链接、打开本地 PDF、下载 PDF
   - 处理流程状态
   - 转文本、分析、导出结果

## 界面原则

- 桌面工具优先，信息密度高于营销感。
- 列表、筛选和操作入口保持稳定，不使用大面积装饰性元素。
- 操作按钮按工作流排布，避免用户在多个页面之间跳转。
- 文档管理要作为一级能力，而不是隐藏在设置里。

## 当前前端原型

当前原型文件：

- `src/main.ts`
- `src/style.css`
- `index.html`

原型包含：

- 年报链接表管理入口
- 年份、板块、行业筛选器
- 年报列表
- 状态统计
- PDF 预览区域
- 下载、打开、转换、分析、导出操作入口

当前原型数据为静态样例，后续将替换为 Tauri command 返回的真实数据。

## 交互状态

### 年报列表

- 默认按更新时间或公司代码排序。
- 点击行后右侧详情区刷新。
- 状态标签展示当前处理阶段。

### PDF 预览

MVP 采用两级策略：

1. 已下载 PDF 优先调用系统默认阅读器打开。
2. 原型界面保留内嵌预览区域，后续接 WebView 本地 PDF 预览。

### 文档管理

文档库包括三类内容：

- 年报链接表 Excel
- 本地 PDF 下载目录
- TXT 和分析结果输出目录

应用需要保存最近使用的文档库配置，避免用户每次重新选择。

## 视觉风格

- 主背景：浅灰工作台背景。
- 侧栏：深色导航与文档管理区。
- 主按钮：低饱和绿色，用于下载、执行等主要动作。
- 强调色：橙红色，用于当前流程节点和品牌标识。
- 表格：紧凑、可扫描、状态清晰。
